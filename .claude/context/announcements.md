# Storyblok SDK Announcement Articles

Published at `storyblok.com/mp/<slug>` to communicate new SDK features and breaking changes.

## Article Types

- **Feature announcement** (minor/patch): new capabilities, DX improvements
- **Major version announcement**: what changed, why, migration path, compatibility

## Structure

**Hook → What Changed → Why It Matters → How to Use It → How to Migrate → Reference → Related Content**

1. **Title** — Feature releases: `[Feature]: [Package Name]`. Major versions: `Introducing [Package Name] v[X]`
2. **Introduction** — 2-3 sentences: what shipped, which package/version, why developers should care
3. **Table of Contents** — Only if 3+ sections
4. **Feature Sections** — One h2 per feature. Each follows: problem → solution → code example
5. **Migration / Breaking Changes** — Before/after code, comparison tables, numbered steps
6. **Compatibility Matrix** — For major versions, version support table
7. **Call-to-Action** — Install command, links to GitHub, npm, docs, Discord
8. **Related Content** — Links to related articles

## Tone & Style

- Professional yet approachable — concise developer communication
- Feature-focused — lead with the benefit, not the implementation
- Prescriptive — guide toward best practices, use "we recommend"
- Reassuring on breaking changes — emphasize backward compatibility

## Formatting Rules

- h2 for major sections, h3 for subsections
- Code blocks with file paths as comments. Multiple package manager options for install commands
- Tables for comparisons, feature matrices, version compatibility
- Bold for key terms on first mention. Inline code for package names, options, function names
- Numbered lists for migration steps. Bullet points for features/benefits
- **No emojis**. **No em dashes** — use commas, colons, or periods instead
